# 撲克勝率（Poker Equity）

德州撲克勝率計算器。輸入你的兩張手牌、目前已開的公共牌（翻牌／轉牌／河牌）和對手人數，用蒙地卡羅模擬 100,000 次即時算出勝率——專為牌局進行中隨手查勝率設計。另內建**練習模式**：翻前決策與勝率估算考題，附 GTO 解說與成績追蹤。

純前端（HTML/CSS/JS），零依賴、可離線，支援 PWA 安裝到 iPhone 主畫面。

## 快速開始

### 方法一：iPhone 直接用（最快）

用手機瀏覽器開啟：

> https://21godman.github.io/holdem/

在 Safari 點「分享 → 加入主畫面」，之後就像 App 一樣點圖示啟動，完全離線可用。

### 方法二：在電腦上本機執行

需要 Node.js（macOS 內建工具即可，無需 npm install）：

```bash
node .claude/serve.js
```

然後開瀏覽器進 http://localhost:8642

### 方法三：部署成正式網站（完整 PWA + 離線）

把整個資料夾上傳到任何靜態網站空間（GitHub Pages、Netlify、Vercel…）即可。
在 HTTPS 網址下 Service Worker 會自動註冊，之後完全離線可用。
本專案即以 GitHub Pages 部署（方法一的網址），push 到 `main` 分支就會自動更新。

## 操作方式

1. **選手牌**：在下方「選牌」區點你的兩張手牌（4 花色 × 13 點數的網格），選完立刻顯示翻牌前勝率。
2. **開公共牌**：翻牌、轉牌、河牌開到哪就點到哪，勝率即時重算。
3. **設對手人數**：點 1–9 的按鈕，隨時可改。
4. **改牌／清除**：點任何已填的牌槽即清除該張牌並重選；「重置牌局」清空全部。
5. **看結果**：
   - **勝率**：純贏（不含平手）的機率
   - **平手**：和至少一位對手平分底池的機率
   - **敗率**：被至少一位對手擊敗的機率
   - **權益（Equity）**：你平均能分到的底池比例（贏 + 平手均分），做決策看這個最準
   - **目前成牌**：翻牌後顯示你現在的牌型（如「兩對」）

已被使用的牌會在選牌區反灰，不能重複選。

## 翻前 GTO 建議

翻牌前的下注建議走內建的 **GTO 圖表查表**（近似公開的 100bb 6-max 現金桌 solver 解，頻率量化到 25%）：

- **首入開池**：前面無人下注（「需跟注」留 0）時，依你的位置查開池（RFI）圖，建議加注尺度與頻率。
- **面對開池**：把「需跟注」填入開池額，並把唯一未棄牌對手的範圍設為對應位置開池（如「CO 開牌」），就會查對應的防守圖，給出含混合頻率的建議（如「3-bet 60%／跟注 40%」）。
- 建議會標明使用了哪張圖表；情境無法對應時（多人底池、對手範圍為任意牌等）自動退回簡化模型並註明。

翻牌後的建議仍為簡化規則模型（勝率對比底池賠率），非 GTO。

## 練習模式

點右上角「練習模式」進入考題介面，適合平時訓練直覺。

### 關卡模式

8 個關卡按主題與難度遞進：勝率入門 → 首入開池 → 面對開池 → 邊界手牌／混合頻率 → 盲位攻防 → 綜合大考。每關 10 題，得分 = 答對數 +0.5×部分正確：

- **≥7 分**過關（★）並解鎖下一關、**≥8.5 分**兩星（★★）、**滿分**三星（★★★），可隨時重打刷星。
- 基礎關只出離決策邊界遠的明確題，進階關全是混合頻率與邊界手牌；勝率題的選項間距也隨難度縮小（16 點 → 8 點）。
- 每關結算有分數、星星與**錯題回顧**（你的答案 vs 正解頻率／實際勝率）。

### 題型與評分

- **翻前決策題**：給你位置、手牌和情境（首入開池或面對某位置的開池），從加注／跟注／棄牌中作答，以內建 GTO 圖表評分。
- **勝率估算題**：給你手牌、牌面和對手人數，從四個選項中估勝率，答案由 50,000 次蒙地卡羅模擬即時算出。
- **混合策略評分**：GTO 頻率 ≥50% 的行動都算正確（50/50 的題兩個答案都對），低頻行動算「部分正確」；答完一律顯示完整頻率分佈。

### 自由練習與成績

- **自由練習**：無限出題（兩種題型隨機混出），18 題精選概念題（阻斷牌、A5s 3-bet 詐唬、BB 防守賠率、set-mine、AA 多人衰減…）穿插出現，附詳細解說。
- **成績追蹤**：關卡進度、星星、題數、正確率、連對都存在裝置上（localStorage）；答錯較多的情境會列在「待加強」面板。可用「重設所有練習紀錄」歸零重來。

## 計算原理

- 對手手牌視為隨機（實戰中你看不到對方的牌），從剩餘牌堆均勻發牌。
- 每次輸入變動即模擬 **100,000 局**：發完對手手牌、補齊公共牌、比大小、統計勝負，誤差約 ±0.3%。
- 模擬跑在 Web Worker（背景執行緒），介面不會卡頓。

## 執行測試

```bash
node test/evaluator.test.js    # 評牌器 + 勝率基準 + GTO 圖表
node test/quiz.test.js         # 練習模式：出題、評分、成績統計、題庫驗證
```

前者包含 89 項測試：全部牌型辨識、牌型大小全序、踢腳與邊界案例（A-5 wheel、三對子踢腳等）、與已知勝率基準的對照（AA 單挑 ≈ 85.2%、72o ≈ 34.6% 等），以及翻前 GTO 圖表的結構驗證與 spot check。後者驗證出題產生器、評分規則與精選題庫（勝率題答案會重新對照模擬結果）。
